使用 Pelican+Nginx 搭建静态 Blog

Wed 15 October 2014

对于个人网站来说，很多情况下一个简单易用的静态Blog就足够了。现在有很多工具可以帮助我们搭建这样的静态Blog，熟悉Ruby的同学可以使用Octopress或者Jekyll，喜欢Python的可以用Pelican，node.js下有Hexo，MetalSmith等等。本文以Pelican+Nginx为例，讲述如何建立一个静态Blog。

安装Pelican

Pelican是一个基于Python的静态网站生成工具，可以简单地使用pip来安装。如果尚未安装Python或者pip，请自行google，安装Pelican时请视需要加上sudo。

pip install pelican markdown

建立静态Blog

首先创建一个目录储存你的Blog

mkdir -p ~/projects/yoursite
cd ~/projects/yoursite

进入目录之后创建一个新的Pelican工程

pelican-quickstart

Pelican会询问一系列配置问题，请按需回答。
此时在yoursite目录下新建了一个名为content的目录，该目录用于存放你的博客文章。Pelican支持富文本，Markdown和AsciiDoc等多种格式，这里以Markdown为例。你是否注意到我们在第一步时，除了Pelican之外还安装markdown，请不要漏掉它，否则Pelican将无法识别出content目录下的Markdown文件。
输入以下文字并将其保存为first.md：

Title: First Blog
Date: 2014-10-15 10:20
Category: Blog

This is my fist Blog. Awesome!

接下去就可以生成静态Blog了，以下两种方式均可，你会注意到命令完成之后，一个完整的静态Blog就生成在output目录下了。 

pelican content

make html

Pelican的配置信息大多存放于pelicanconf.py和publishconf.py两个文件中，请参考Pelican 文档。

Pelican的默认主题并不是特别美观，我们也可以自行创建主题或者使用网上开源的主题，例如pelican-bootstrap3。

mkdir -p theme
cd theme
git clone https://github.com/DandyDev/pelican-bootstrap3.git
pelican content -t theme/pelican-bootstrap3

如果你不想每次生成Blog的时候手动指定主题，也可以将主题加入配置文件pelicanconf.py。

THEME = "theme/pelican-bootstrap3"

部署到Nginx

至此，我们已经有了静态Blog的所需的所有文件，但为了能在互联网上访问，我们还需要将其部署到Web服务器上。现在许多网站均支持静态站点托管，例如Dropbox，Google Drive，Github等。国内的有Aliyun OSS，GitCafe等。如果想自行搭建Web服务器，可以使用Nginx。
Nginx可以通过apt-get来安装：

apt-get update
apt-get install nginx

顺利安装完Nginx之后，我们需要修改其配置文件/etc/nginx/nginx.conf，首先留好备份。

sudo cp /etc/nginx/nginx.conf /etc/nginx/nginx.conf.bak

在/etc/nginx/nginx.conf中加入：

http {
    server {
        server_name www.example.com;
        listen 80;

        location / {
            root /home/xxx/projects/yoursite;
        }
    }
}

这是一个简单的例子，静态Blog将被部署到www.example.com，其中server_name是站点的域名，如果没有域名可以使用IP地址代替，location的root参数指向静态Blog在服务器中的位置。
重启Nginx，就可以访问你的Blog了。

sudo service nginx restart

至此，一个简单的静态Blog就搭建完毕了，其它工具或者平台略有差异，但基本思路是一致的。
